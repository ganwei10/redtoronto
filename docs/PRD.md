## 2. 产品概述

红多营运营平台是面向**多伦多小红书营销**的一站式智能运营 SaaS：连接**商户、博主、运营方**三类角色，将「需求录入 → 博主匹配 → 内容创作 → 投放（代理商侧）→ 效果归因 → AI 复盘」整合成可运营、可复制的闭环。

- **一句话定位**：商户提需求 → AI 出 Brief → 系统匹配博主 → AI 出内容 → 代理商合规投放 → 看板归因 → AI 复盘沉淀方法论。
- **技术基线**：Next.js 15（App Router）+ TypeScript + Tailwind；PostgreSQL + pgvector；Anthropic Claude；Vercel 部署；NextAuth / JWT 统一身份认证。

---

## 3. 用户角色与权限

平台区分四类角色，其中**商户 / 运营方 / 博主**拥有独立登录入口与操作台。

| 角色 | 独立登录 | 核心权限 |
|---|---|---|
| 运营方 Operator | ✅ `/login` 选「运营方」 | 全部功能：博主库、匹配、内容、Campaign、看板、知识库、商户管理 |
| 商户 Merchant | ✅ `/login` 选「商户」 | 提交需求 Brief、查看匹配推荐、查看 Campaign 效果、管理合作 |
| 博主 Creator | ✅ `/login` 选「博主」 | 维护主页/报价/接单状态、接收派单、提交内容稿、查看收益 |
| 代理商 Agent | （只读后台，可选） | 投放/结算数据只读，依赖代理商开放接口 |

> **关于「三角色独立登录」放在哪个版本？**
> 旧版 PRD（v0.1–v0.4）仅在角色表中列出三类角色，但**没有登录功能，也没有版本归属**。
> 本版将其明确纳入路线图：**v1.0 上线商户门户（商户注册/登录 + 需求提交 + 效果查看）**，
> **v1.1 上线博主门户并完成三角色统一身份认证与权限（NextAuth / JWT + 数据库，RBAC）**。
> 当前 v0.1–v0.4 的代码为内部运营工具（运营方视角），已内置 `/login` 角色入口演示（前端角色态），
> 真实账号体系在 v1.1 落地。

## 4. 目标与范围

**目标**：从「内部运营工具」升级为「多角色可运营平台」，形成可复制的多伦多本地营销方法论，并为规模化（多城市、代理商结算）打好基础。

**本期（v2.0 规划）范围**
- 三角色账户体系与权限（RBAC）
- 商户门户：需求提交、推荐查看、效果看板
- 博主门户：主页维护、派单接收、交稿、收益查看
- 运营中台：博主库 / 匹配 / 内容 / Campaign / 看板 / 知识库（已具备）
- 全链路数据校验与合规（已具备基础）

**后续范围（Out）**
- 支付/结算自动化、蒲公英投放 API 深度对接、多语言全套、多城市、代理商分润后台。

## 5. 功能需求

### FR1 · 博主库管理（运营方）
- 录入/编辑博主：`handle, platformId, followers, engagementRate, niche[], city, rateCAD, availability, pastCases[]`
- 列表 / 筛选 / 搜索；标签体系（赛道 / 调性 / 粉丝层级）
- **参数校验**：粉丝量 ≥ 0 整数、互动率 0–100（%）、报价 ≥ 0、账号必填；支持 CSV/JSON 批量导入并自动去重、拦截非法值。

### FR2 · 需求 Brief 录入与 AI 生成（商户 / 运营方）
- 商户表单：行业、目标、预算、目标人群、卖点、KPI、周期
- AI（Claude）生成结构化 Brief：人群包、内容方向、推荐博主类型、KPI 拆解
- **参数校验**：行业必填、预算 ≥ 0。

### FR3 · 博主匹配
- 输入 Brief / 行业 + 预算 + 城市 → 综合打分（赛道重合、互动率、报价匹配、接单状态）→ Top-N 推荐 + 理由
- **参数校验**：预算 ≥ 0、返回数量 1–50。

### FR4 · 内容创作工作台
- 选博主 + Brief → AI 生成种草文案 / 标题 / 选题（多版本）
- 中英双语本地化；合规词表校验（极限词 / 广告法 / 合作披露）
- 内容状态机：`草稿 → 待审 → 已发 → 复盘`

### FR5 · Campaign 与效果看板
- 创建 Campaign，绑定 Brief / 博主 / 内容
- 指标录入：曝光、互动、点击、留资（**均 ≥ 0 整数**，原 `prompt` 弹窗已改为内联校验表单）
- 看板：曝光 / 互动 / 留资 / 单留资成本 ROI；异常预警

### FR6 · AI 复盘报告
- Campaign 结束 → AI 生成复盘（亮点 / 问题 / 下一轮建议）
- 报告沉淀进知识库，供 RAG 调用

### FR7 · 三角色账户与权限（v1.1，平台化核心）
- 统一登录入口 `/login`：商户 / 运营方 / 博主三类独立身份
- 认证：NextAuth（Auth.js）/ JWT + 数据库（PostgreSQL）
- 权限模型 RBAC：角色 → 资源 → 操作；数据按租户/角色隔离
- 密码策略、找回、会话管理、审计日志

### FR8 · 商户门户（v1.0）
- 商户注册 / 登录、企业资料维护
- 需求 Brief 提交与历史
- 查看匹配推荐、Campaign 效果看板、合作记录

### FR9 · 博主门户（v1.1）
- 博主注册 / 登录、个人主页（粉丝、报价、赛道、案例）
- 接单状态维护、派单接收与确认
- 内容稿提交与状态跟踪、收益（预估/结算）查看

## 6. 非功能需求

- **性能**：博主库 < 1k 条时，列表/匹配响应 < 1s；页面首屏 < 2s（Vercel Edge）。
- **安全**：密钥仅在服务端（Vercel 函数）；商户/博主 PII 加密存储；RBAC 最小权限。
- **数据校验**：所有用户输入前后端双重校验，拦截负数、NaN、越界、必填缺失（见 FR1–FR6）。
- **合规**：付费合作仅走蒲公英；合作披露强制（material connection）；遵循加拿大 PIPEDA，留资最小化。
- **可观测**：错误日志、AI 调用用量统计、限流；embedding 失败回退标签打分。

## 7. 验收标准

- [ ] 三类角色均可通过 `/login` 独立登录并进入对应操作台（v1.1）
- [ ] 博主库可录入 ≥ 30 位博主，负数/非法值被拦截，批量导入去重生效
- [ ] Brief AI 生成可用，匹配 Top-5 合理且带理由
- [ ] 效果指标录入为内联校验表单，负数无法提交
- [ ] 看板可展示核心指标；复盘可由 AI 生成并沉淀知识库
- [ ] 不同角色仅能访问授权资源（RBAC）
